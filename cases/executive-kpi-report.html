The Data Challenge (2016)

Every week, the General Manager required a consolidated view of the business to make strategic decisions. However, the data landscape was fragmented. Information arrived in isolated "silos" via email, often with conflicting metrics.

My responsibility was to centralize over 20+ distinct data sources into a master report, ensuring that Finance (EBITDA), Network (Availability), and Commercial (Churn) numbers were consistent.

Technical Implementation (Code Snippet)

Instead of VLOOKUPs, we use SQL for robust transformations. Here is how I would model the "Customer Churn" logic using BigQuery SQL:

process_daily_kpis.sql
MERGE `claro-data-warehouse.exec_reports.daily_kpis` T
USING (
    SELECT
        date_id,
        region,
        SUM(gross_adds) AS total_activations,
        SUM(churn_vol) + SUM(churn_invol) AS total_churn,
        -- Calculated Field: Net Adds
        (SUM(gross_adds) - (SUM(churn_vol) + SUM(churn_invol))) AS net_adds
    FROM `claro-data-warehouse.staging.commercial_movements`
    WHERE date_id = CURRENT_DATE()
    GROUP BY 1, 2
) S
ON T.date_id = S.date_id AND T.region = S.region
WHEN MATCHED THEN
    UPDATE SET total_churn = S.total_churn, net_adds = S.net_adds
WHEN NOT MATCHED THEN
    INSERT (date_id, region, total_activations, total_churn, net_adds)
    VALUES (S.date_id, S.region, S.total_activations, S.total_churn, S.net_adds);
