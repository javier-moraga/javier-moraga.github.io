Legacy to Cloud: SAS/Oracle to BigQuery

Modernizing the data infrastructure of a top-tier healthcare provider (Clínica Alemana) handling tables up to 90M+ records.

Legacy Stack SAS / Oracle / On-Prem
Modern Stack GCP / BigQuery / Airflow
Data Volume 90M+ Rows / Table
Key Focus Governance & Security

The Challenge: Siloed & Slow Analytics

Decision-making processes were reliant on legacy SAS Enterprise Guide flows connected to on-premise Oracle databases. These workflows were becoming difficult to maintain, lacked version control, and struggled to process growing datasets efficiently.

Original State: Complex, manual SAS flows with limited observability.

The Solution: Automated "Data Ingestor" Framework

We engineered a custom Python framework deployed on **Kubernetes (GKE)** and orchestrated by **Airflow**. This "Data Ingestor" automates the ELT (Extract, Load, Transform) process, moving data securely from on-premise databases to the Data Lake.

Oracle / Postgres On-Prem Source
Data Ingestor Python / Docker / VPN
BigQuery RAW String Ingestion
Transformation SQL Casting & Quality

Data Governance & Security by Design

Handling sensitive patient data requires strict governance. The migration didn't just move data; it categorized it. We implemented a strict Labeling Standard directly in the DDL of BigQuery tables.

PII Protection Tables containing Patient ID or personal info are tagged with dg-has-pii: si and dg-data-classification: altamente_sensible.
Automated Quality The framework automatically counts cast_error_count and flags suspicious_dates (e.g., year < 1900) during ingestion.
bigquery/ddl/create_patient_table.sql SQL
CREATE TABLE `{project-id}.ds_raw_datalake.pacientes` ( id_paciente STRING OPTIONS(description = "Identificador único"), fecha_ingreso DATE OPTIONS(description = "Fecha registro sistema"), diagnostico STRING OPTIONS(description = "Código CIE-10") ) PARTITION BY DATE(fecha_ingreso) CLUSTER BY id_paciente OPTIONS( description = "Tabla maestra de pacientes con datos sensibles", labels = [ ("dg-layer", "raw"), ("dg-ingestion-type", "fundacional"), ("dg-has-pii", "si"), ("dg-data-classification", "altamente_sensible"), ("dg-domain", "salud") ] );
